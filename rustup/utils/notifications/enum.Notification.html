RenameInUse(&'a Path, &'a Path)

Renaming encountered a file in use error and is retrying. The InUse aspect is a heuristic - the OS specifies Permission denied, but as we work in users home dirs and running programs like virus scanner are known to cause this the heuristic is quite good.
